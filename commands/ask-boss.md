# /ask-boss - 子代理請求老闆決策

> 任何代理在需要老闆授權或決策時，透過大腦呼叫此 skill。
> 禁止子代理直接打擾老闆，所有升級都必須經過大腦過濾。

## 輸入

$ARGUMENTS（格式：`{代理名稱}|{問題類型}|{問題描述}`）

例如：
- `BACKEND|技術決策|需要選擇 Redis 或 Memcached 作為 session cache`
- `SA|架構衝突|BACKEND 認為應用 GraphQL 但原本規劃是 REST`
- `FRONTEND|權限請求|需要存取 .env.local 取得 API key`

---

## 執行流程

### Step 1：大腦先評估

大腦收到請求後，先判斷：

**是否真的需要老闆決策？**

```
判斷標準：

A. 大腦可以自行決定的情況（直接決定，不打擾老闆）：
   - 技術細節（Redis vs Memcached → SA 或大腦決定）
   - 代理間的小衝突（不影響整體架構）
   - 明顯有標準答案的問題

B. 必須問老闆的情況：
   - 等級 3 權限請求（付費 API、機敏資訊、工作區以外路徑）
   - 影響整體方向的決策（更換技術棧、大幅更改需求）
   - Sprint 時程大幅延遲（超過原估計 50%）
   - 有法律/安全/成本風險的決策
   - 大腦自己也不確定的重要問題
```

### Step 2A：若大腦可以決定

直接回覆提出請求的代理，不打擾老闆。
記錄決策到 `.ai-team/plans/decisions.md`：
```
{日期} {代理} 的問題：{問題描述}
大腦決定：{決定內容}
理由：{理由}
```

### Step 2B：若必須問老闆

**在 Claude Code 介面顯示：**

```
🔴 [需要你的決策]

提出者：{代理名稱}
問題：{問題描述}

背景：{大腦補充的 context}

選項：
A. {選項A} → 優點：{} 缺點：{}
B. {選項B} → 優點：{} 缺點：{}
（若是權限請求）確認執行 / 取消

影響：{若做這個決定，後續會怎樣}

你的決定？
```

**同時發 Telegram：**
```
[BRAIN] 🔴 需要你的決策

{問題一行摘要}
{2行背景}

選項：
A. {選項A}
B. {選項B}
（若權限請求）✅ 確認 | ❌ 取消

📎 完整資訊在 Claude Code
```

### Step 3：收到老闆決定後

1. 記錄決策到 `.ai-team/plans/decisions.md`
2. 通知相關代理，讓其繼續工作
3. 若需要調整 Sprint 計畫，更新 PM 的 Backlog

---

## 注意事項

- 大腦應該**過濾掉至少 80% 的問題**，自行決定，不要每件小事都問老闆
- 問老闆時，必須提供選項和影響分析，不要只丟問題
- 緊急問題（安全漏洞、資料遺失風險）優先級最高，立即通知
